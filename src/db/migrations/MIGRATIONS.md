# Миграции базы данных

## 📋 Описание

Миграции используются для управления изменениями в структуре базы данных SQLite. Каждая миграция имеет функции `up()` (применить) и `down()` (откатить).

## 🗂 Структура

```
src/db/migrations/
└── removeTokenUniqueConstraint.ts
```

## 🚀 Запуск миграций

### Применение миграции (up)

Чтобы применить миграцию и удалить ограничение UNIQUE с поля `tokenGitLab`:

```bash
npx tsx src/db/migrations/removeTokenUniqueConstraint.ts up
```

### Откат миграции (down)

Чтобы откатить миграцию и вернуть ограничение UNIQUE:

```bash
npx tsx src/db/migrations/removeTokenUniqueConstraint.ts down
```

## 🔧 Доступные миграции

### removeTokenUniqueConstraint

**Дата:** 2026-03-15  
**Описание:** Удаляет ограничение UNIQUE с поля `tokenGitLab` в таблице `chatConfig`

**До миграции:**

- Поле `tokenGitLab` не может повторяться в разных чатах
- Каждый чат должен иметь уникальный токен GitLab

**После миграции:**

- Поле `tokenGitLab` может повторяться
- Несколько чатов могут использовать один и тот же токен GitLab

**Что делает миграция:**

1. Создаёт резервную копию базы данных (`data/late.sqlite.backup`)
2. Создаёт новую таблицу `chatConfig_new` без ограничения UNIQUE на `tokenGitLab`
3. Копирует все данные из старой таблицы в новую
4. Удаляет старую таблицу `chatConfig`
5. Переименовывает новую таблицу в `chatConfig`
6. Логирует результат через pino logger

**Безопасность:**

- Перед миграцией автоматически создаётся бэкап базы данных
- Все операции выполняются в транзакции (атомарность)
- При ошибке происходит откат транзакции

## 📝 Создание новой миграции

Для создания новой миграции:

1. Создайте файл в `src/db/migrations/` с названием миграции
2. Экспортируйте функции `up()` и `down()`
3. Используйте транзакции для безопасности

```typescript
export async function up(): Promise<void> {
  // Код применения миграции
}

export async function down(): Promise<void> {
  // Код отката миграции
}

if (require.main === module) {
  const command = process.argv[2];

  if (command === 'up') {
    up()
      .then(() => {
        logger.info('Миграция вверх успешно завершена');
        process.exit(0);
      })
      .catch((err) => {
        logger.error({ err }, 'Ошибка миграции вверх');
        process.exit(1);
      });
  } else if (command === 'down') {
    down()
      .then(() => {
        logger.info('Миграция вниз успешно завершена');
        process.exit(0);
      })
      .catch((err) => {
        logger.error({ err }, 'Ошибка миграции вниз');
        process.exit(1);
      });
  }
}
```

## 🔒 Резервные копии

При выполнении миграции автоматически создаётся бэкап базы данных:

- `data/late.sqlite` — основная база данных
- `data/late.sqlite.backup` — резервная копия

**Важно:** Если миграция не удалась, можно восстановить базу из бэкапа.

## 📊 Логирование

Все операции миграций логируются через pino logger в файл `logs/bot.log`:

- Информационные сообщения (успешное выполнение)
- Сообщения об ошибках (с детальным контекстом)

4. **Храните историю миграций для возможного отката**
