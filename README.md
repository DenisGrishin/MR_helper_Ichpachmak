# MR-Helper Bot

Telegram бот для интеграции с GitLab и управления Merge Requests (MR) в групповых чатах.

## 📋 Описание

MR-Helper — это интеллектуальный Telegram бот, который помогает командам отслеживать Merge Requests из GitLab. Бот автоматически собирает информацию о MR, управляет списком выполненных задач и предоставляет удобный интерфейс для администрирования.

## ✨ Основные возможности

### Отслеживание Merge Requests

- **Автоматический сбор MR** — отправьте ссылку на MR с префиксом `!!` для автоматического отслеживания
- **Пресетные MR** — отправьте ссылку с префиксом `!` для быстрого добавления в пресет
- **Интеграция с GitLab API** — автоматическое получение информации о статусах, авторах и reviewer'ах

### Управление пользователями

- Добавление пользователей в проектные чаты
- Редактирование пресетов для пользователей
- Удаление пользователей из чатов
- Управление статусом отправки MR

### Управление чатами

- Настройка конфигурации для каждого чата
- Добавление GitLab токенов для проектов
- Список выполненных задач по чатам
- Автоматический сброс задач в полночь

### Команды бота

- `/menu` — открыть главное меню
- `/setUser` — добавить пользователя в чат проекта
- `/completedTasks` — просмотр выполненных задач

### Интерактивное меню

- Панель администратора с полным доступом
- Панель пользователя с ограниченным функционалом
- Управление настройками чатов
- Просмотр списков пользователей

## 🛠 Технологии

- **Node.js** (>=22.2.0)
- **TypeScript** — типизация и безопасность кода
- **Grammy** — современный фреймворк для Telegram ботов
- **SQLite3** — встроенная база данных
- **Axios** — HTTP клиент для работы с GitLab API
- **Node-cron** — планировщик задач

## 📦 Установка

### Клонирование репозитория

```bash
git clone https://gitlab.devds.ru/bots/mr-helper.git
cd mr-helper
```

### Установка зависимостей

```bash
npm install
```

### Настройка переменных окружения

Создайте файл `.env` в корне проекта на основе `.env.example`:

```env
BOT_API_KEY=your_telegram_bot_token_here
BASE_URL=gitlab.com
```

Где:

- `BOT_API_KEY` — токен вашего Telegram бота (получить у [@BotFather](https://t.me/BotFather))
- `BASE_URL` — URL вашего GitLab инстанса (например, `gitlab.com` или `git.russpass.dev`)

## 💡 Использование

### Добавление бота в чат

1. Добавьте бота в групповой чат Telegram
2. Сделайте бота администратором чата (для доступа к сообщениям)
3. Настройте конфигурацию чата через меню администратора

### Отслеживание MR

Отправьте ссылку на MR в чат с одним из префиксов:

- `!!https://gitlab.com/project/repo/-/merge_requests/123` — автоматическое отслеживание
- `!https://gitlab.com/project/repo/-/merge_requests/123` — добавление в пресет

### Управление пользователями

1. Напишите боту `/menu` в личных сообщениях
2. Выберите соответствующий пункт в меню администратора
3. Выберите чат проекта
4. Выполните необходимое действие

### Удаление сообщений бота

Для удаления сообщений бота в чате используйте команду:

```
del-msg-bot
```

## 🔧 Настройка

### Добавление GitLab токена

1. Получите Personal Access Token в GitLab (Settings → Access Tokens)
2. Добавьте бота в чат
3. Настройте конфигурацию через меню администратора
4. Укажите токен в настройках чата

### Настройка администраторов

Администраторы бота настраиваются в коде в файле `src/helper/helper.ts`:

```typescript
export function isAdminUser(userId: number): boolean {
  // Добавьте ID администраторов
  return [123456789, 987654321].includes(userId);
}
```

## 📊 База данных

Проект использует SQLite для хранения данных. Файл базы данных создается автоматически в директории `data/`.

Основные таблицы:

- `chats_config` — конфигурация чатов
- `chat_members` — участники чатов
- `users` — пользователи системы
- `completed_tasks` — выполненные задачи

## 🐛 Обработка ошибок

Бот имеет встроенный наблюдатель за ошибками (ErrorObserver), который логирует все ошибки и предотвращает падение бота.
